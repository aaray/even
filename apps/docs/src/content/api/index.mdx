# API Reference

The EVEN API provides programmatic access to the platform's data and functionality.

## Base URL

```
http://localhost:4000
```

In production, replace with your actual API endpoint.

## Authentication

Currently, the API is unauthenticated for development purposes. Production deployments will require authentication.

## Response Format

All responses are JSON formatted:

```json
{
  "data": { ... },
  "meta": {
    "requestId": "req-abc123",
    "timestamp": "2026-02-05T10:30:00Z"
  }
}
```

### Error Responses

Errors follow a consistent format:

```json
{
  "error": "Not Found",
  "message": "Resource not found",
  "statusCode": 404,
  "requestId": "req-abc123"
}
```

## Available Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| [`/health`](/api/endpoints/health) | GET | Health check |
| [`/artist`](/api/endpoints/artist) | GET | Artist profile |
| [`/products`](/api/endpoints/products) | GET | Product listing |
| [`/earnings`](/api/endpoints/earnings) | GET | Earnings data |
| [`/fans`](/api/endpoints/fans) | GET | Fan metrics |

## Quick Examples

### Get Artist Profile

```bash
curl http://localhost:4000/artist
```

### List Products with Filtering

```bash
curl "http://localhost:4000/products?category=music&sort=revenue"
```

### Get Earnings for Last 30 Days

```bash
curl "http://localhost:4000/earnings?range=30d"
```

## Rate Limiting

Development servers have no rate limiting. Production APIs are rate-limited to:
- 100 requests per minute per IP
- 1000 requests per hour per authenticated user
