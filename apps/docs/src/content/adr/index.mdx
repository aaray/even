# Architecture Decision Records

Architecture Decision Records (ADRs) capture important architectural decisions made during the development of EVEN. Each ADR documents the context, decision, and consequences to help current and future team members understand why things are built the way they are.

## ADR Status Legend

| Status | Meaning |
|--------|---------|
| **Proposed** | Under discussion, not yet decided |
| **Accepted** | Decision made and in effect |
| **Deprecated** | No longer recommended, but not replaced |
| **Superseded** | Replaced by a newer ADR |

## Creating New ADRs

To create a new ADR:

1. Copy the [ADR template](/adr/how-to-create)
2. Follow the naming convention: `NNN-kebab-case-title.mdx`
3. Fill in all required sections
4. Submit for review

See the [How to Create an ADR](/adr/how-to-create) guide for detailed instructions.
